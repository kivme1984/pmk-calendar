# PMK Calendar release rules

Этот файл фиксирует обязательные правила, чтобы не отдавать пользователю старые версии и не запускать цикличные обновления.

## 1. Тестовые страницы не являются обновлением

Любой макет, визуальный тест или пример интерфейса должен открываться отдельной HTML-страницей и не должен менять рабочий `index.html`, `sw.js`, `sw-core*`, `version.json` и `pmk-release.json`.

Для тестов использовать whitelist-страницы:

- `safe.html?test=...`
- `reset.html?...` только для реального обновления
- `recovery.html?...` только для восстановления
- отдельные `*-test*.html` / `preview-*.html`, которые должны быть пропущены через `sw-special-router-v82.js`

## 2. Запрещено давать ссылку, которую service worker может подменить на старый index.html

Если создаётся тестовая HTML-страница, она обязана быть добавлена в `sw-special-router-v82.js` или открываться через `safe.html`, иначе старый service worker может показать старую рабочую версию.

## 3. При реальном обновлении версии синхронизировать все файлы сразу

Перед публикацией обновления должны совпадать:

- `sw.js`
- активный `sw-core-*.js`
- `update-manager-v82-20.js`
- `version.json`
- `pmk-release.json`
- `reset.html`

Нельзя поднимать версию только в одном файле. Это вызывает цикличное окно обновления.

## 4. Автоматическое окно обновления отключено

`update-manager-v82-20.js` не должен сам показывать бесконечные модальные окна обновления. Обновление запускается только вручную через кнопку или reset-ссылку.

## 5. Сначала тестовая страница, потом рабочее обновление

Порядок работы:

1. Создать отдельную тестовую страницу.
2. Проверить, что она открывается именно как тестовая страница, а не как старый календарь.
3. Получить подтверждение пользователя.
4. Только после этого переносить в рабочую сборку.

## 6. Контроль перед отправкой ссылки пользователю

Перед тем как отправить ссылку пользователю, проверить:

- файл существует в репозитории;
- `sw-special-router-v82.js` пропускает эту страницу напрямую;
- ссылка содержит cache-bust параметр `?v=...` или `?preview=...`;
- ссылка не ведёт на рабочий `/` вместо тестовой страницы.
