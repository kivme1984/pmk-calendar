# Подключение Яндекс.Календаря к ПМК Календарю

Версия интеграции: v69 (предварительное внедрение).

## Архитектура

- приложение остаётся на GitHub Pages;
- Google Calendar продолжает работать без изменений;
- Яндекс.Календарь подключается через тот же Cloudflare Worker, который обслуживает адресные подсказки;
- логин Яндекса и пароль приложения хранятся только в секретах Cloudflare;
- браузер передаёт Worker отдельный `PMK_SYNC_TOKEN`;
- заявки сопоставляются между календарями по постоянному `pmkId`;
- если один календарь недоступен, операция сохраняется в локальную очередь и повторяется после восстановления связи.

## 1. Создать отдельный календарь в Яндексе

Рекомендуемое имя: `ПМК — Заявки`.

В настройках этого календаря скопируйте его точный CalDAV URL. Нужна ссылка коллекции календаря, а не публичная ссылка iCal для чтения.

## 2. Создать пароль приложения Яндекса

В Яндекс ID создайте отдельный пароль приложения типа «Календарь». Обычный пароль аккаунта не используйте.

## 3. Обновить Cloudflare Worker

В существующем Worker замените код содержимым файла:

`cloudflare-worker-pmk-services-v1.js`

Код сохраняет действующий маршрут `/suggest` для DaData и добавляет маршруты `/calendar/...`.

## 4. Добавить секреты Worker

В Cloudflare: Worker → Settings → Variables and Secrets.

Добавьте:

- `DADATA_API_KEY` — уже используемый ключ адресных подсказок;
- `YANDEX_CALDAV_URL` — точный CalDAV URL календаря `ПМК — Заявки`;
- `YANDEX_LOGIN` — логин или полный адрес почты Яндекса;
- `YANDEX_APP_PASSWORD` — пароль приложения «Календарь»;
- `PMK_SYNC_TOKEN` — отдельная длинная случайная строка, минимум 24 символа;
- `ALLOWED_ORIGIN` — `https://kivme1984.github.io`.

После сохранения выполните Deploy.

## 5. Настроить приложение ПМК

После обновления до v69 откройте:

`Настройки → Яндекс.Календарь`

Укажите:

- адрес Worker: `https://lucky-math-8e63pmk-address.standart-media.workers.dev/calendar`;
- тот же `PMK_SYNC_TOKEN`, который добавлен в Cloudflare;
- включите «Дублировать заявки в Яндекс»;
- нажмите «Сохранить настройки»;
- нажмите «Проверить Яндекс».

Логин Яндекса и пароль приложения в приложение вводить не нужно.

## 6. Перенести существующие заявки

После успешной проверки нажмите:

`Перенести заявки из Google`

Приложение перенесёт только структурированные события ПМК, в которых сохранён `pmkId`. Повторный запуск не создаёт копии: события обновляются по тому же идентификатору.

## 7. Проверка

Создайте тестовую заявку и убедитесь, что она появилась:

1. в Google Calendar;
2. в Яндекс.Календаре;
3. в ПМК Календаре после ручной синхронизации.

Затем измените время, статус и номер договора, после чего удалите тестовую заявку. Все четыре операции должны повториться в обоих календарях.

## Важно

`PMK_SYNC_TOKEN` не является паролем Яндекса. Это отдельный технический ключ доступа к вашему Worker. Его нужно вводить только на рабочих устройствах менеджеров.
